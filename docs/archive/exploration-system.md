# Exploration System (Archived)

**Status**: Archived — not being developed for the revival. The code remains in `exploration/` but is not a focus.

## What It Was

The exploration system let players send monsters on solo expeditions outside the ring. A monster would be dispatched and, after a delay, return with one of several outcomes:

- Nothing (safe return)
- A hazard (damage, debuff)
- A death card (monster dies — permanent)
- Loot (items, coins)
- A new card for their deck

The intent was to give players something to do with monsters that weren't in the ring, and to add a sense of the world being bigger than the arena.

## Why It's Archived

The feature was never heavily used compared to the ring combat. The revival is focused on making the core loop great — collect monsters, build decks, fight in the ring. Exploration can be reconsidered once the core is solid.

## Potential Revival

If brought back, the exploration system would benefit from:
- More varied outcomes with meaningful narrative flavor
- Tie-in to the multi-room concept (explore a specific "world" tied to a room)
- Visual map or zone concept for the web/mobile apps
- Integration with the card economy (exploration as a card acquisition path)

## Code Location

`exploration/` in the engine root. The code is intact and functional as of the last JS version.
